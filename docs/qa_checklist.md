# TCG QA Checklist (functional)

## 1) 매치 종료/방 재사용
- [ ] 2인 매치 시작 후 항복/승리로 종료한다.
- [ ] 종료 직후 로비로 이동 시 같은 방 코드가 유지된다.
- [ ] 같은 방으로 다시 입장하면 새 매치가 시작된다.
- [ ] 로비↔게임 자동 순환(리다이렉트 루프)이 발생하지 않는다.

## 2) 새로고침/복구
- [ ] 로비에서 새로고침해도 roomId가 유지된다.
- [ ] 게임에서 새로고침해도 roomId/agentId가 복구된다.
- [ ] URL에서 query 일부가 빠져도 sessionStorage 기반 복구가 동작한다.

## 3) 턴/권한 가드
- [ ] 상대 턴에 `play_card`/`attack` 시 서버가 차단한다.
- [ ] 배틀 페이즈가 아닐 때 공격이 차단된다.
- [ ] actorId를 클라이언트에서 주입해도 서버가 거부한다.

## 4) 전개/전투
- [ ] 유닛/마법 전개가 각 존 규칙대로 동작한다.
- [ ] 유닛↔유닛 전투 후 사망 처리/무덤 이동이 정상이다.
- [ ] 에이전트 직접 공격 시 HP 감소/승패 판정이 정상이다.

## 5) 스택/효과
- [ ] `direct_hit` 사용 시 스택 적재/해결(3 피해)이 정상이다.
- [ ] 효과 비용(`cost.mana`) 부족 시 해당 효과가 스킵되고 게임은 계속 진행된다.
- [ ] 턴당 1회 제한 효과가 중복 발동되지 않는다.

## 6) 덱 경계
- [ ] 30장 미만 저장 불가.
- [ ] 동명 카드 3장 초과 불가.
- [ ] 저장 후 불러오기 결과가 일치한다.

## 7) 관전/비참가자
- [ ] 3번째 유저 입장 시 관전 또는 제한 정책이 일관되다.
- [ ] 비참가자 `rooms state` 응답에 민감 game 상태가 노출되지 않는다.

## 최근 회귀 이슈 메모
- 방은 재사용 가능해야 하며, 매치 종료 시 `room.game`만 초기화한다.
- 존 하이라이트/깜빡임이 동작하지 않아야 한다.
- 카드는 손패/존 모두 카드명·효과·(유닛이면)공격/체력을 표시한다.
