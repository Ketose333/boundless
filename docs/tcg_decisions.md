# TCG Decisions (single source)

이 문서는 "왜 이렇게 결정했는가"를 기록한다.  
세부 룰 본문은 [`rules.md`](./rules.md) 단일본을 따른다.

## 확정

- 목표: 에이전트 vs 에이전트 대전이 기본이며, AI 에이전트는 옵션 어댑터로 붙인다.
- 핵심 원칙: 게임 서버/룰엔진은 AI 서비스가 꺼져도 정상 동작해야 한다.
- OpenClaw 비의존 원칙: 코어/서버는 OpenClaw 없이 단독 실행 가능해야 한다.
- MVP 우선순위: 룰엔진 인터페이스 → 턴 루프 → 액션 검증 → 상태 동기화.
- 공식 룰 단일본: [`rules.md`](./rules.md)

## 미확정

- 게임 공식 이름 (기존: 바운드리스)
- 카드 타입 최소셋 (유닛/마법 세부)
- Stack 처리 우선순위 상세

## 보류

- 시즌/이벤트/칭호 시스템
- 관전 모드
- AI 전용 난이도 프리셋

## 구현 메모 (Alpha 1.0.0)

- 서버 authoritative 유지
- 명칭/변수 규칙을 최신 사양으로 통일
- API는 Vercel Hobby 한도 대응을 위해 엔트리 수 최소화
- 인게임 종료 직후 `lobby ↔ game` 자동진입 루프 방지 쿨다운 적용

## UI/표현 기준 상태

- 상단 상태 HUD 단일 섹션 기준 유지
- 존/손패 카드 정보 표시는 카드명·효과·(유닛)공격/체력으로 통일
- 문체 기준: 일반 UI 문구는 해요체를 기본으로 유지
- 키워드 설명 문구는 예외적으로 서술형 `~다.` 종결로 통일

## 변경 규칙

- 이 문서는 append/update로만 관리한다.
- 구현 전에 결정 문서를 먼저 갱신한다.
- 룰 본문 수정은 반드시 [`rules.md`](./rules.md)에서 먼저 수행한다.
